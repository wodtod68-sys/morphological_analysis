# morphological_analysis
# Korean POS Corpus — Morpheme & Index Term Extractor

형태소 분석 결과(POS tagged corpus)로부터 형태소/품사를 추출하고,
색인어(단일어·복합어)를 생성하는 두 개의 Python 스크립트입니다.

---

## 파일 구성

| 파일 | 역할 |
|------|------|
| `get_morphs_tags.py` | 형태소 분석 파일 → 형태소/품사 목록 추출 |
| `tagged2context.py`  | 형태소 분석 파일 → 색인어 문맥 파일 생성 |

---

## 실행 방법

### get_morphs_tags.py

```bash
./get_morphs_tags.py <입력파일> > <출력파일>
```

**예시**
```bash
./get_morphs_tags.py all.tag > all.mt
```

- 입력: 탭으로 구분된 2컬럼 형식 (`어절\t형태소분석결과`)
- 출력: 탭으로 구분된 2컬럼 형식 (`형태소\t품사`)

---

### tagged2context.py

```bash
./tagged2context.py <입력파일(s)>
```

**예시**
```bash
./tagged2context.py all.tag
./tagged2context.py *.tag        # 여러 파일 동시 처리
```

- 입력: 탭으로 구분된 2컬럼 형식 (`어절\t형태소분석결과`)
- 출력: 입력파일명 + `.context` (예: `all.tag` → `all.tag.context`)
- 각 문장은 공백으로 구분된 색인어 한 줄로 출력되며, 문장 경계는 빈 줄로 구분

---

## 입력 파일 형식

각 줄은 `어절\t형태소분석결과` 형식이며, 빈 줄은 문장 경계를 나타냅니다.

형태소 분석 결과의 기본 형식은 `형태소/품사+형태소/품사+...`입니다.

```
김현욱(金顯煜)사무총장도    김현욱/NNP+(/SS+金顯煜/SH+)/SS+사무총장/NNG+도/JX
최근                        최근/NNG
합의정신에                  합의정신/NNG+에/JKB
                            ← 빈 줄: 문장 경계
```

## get_morphs_tags.py 상세 설명

### 핵심 함수: `get_morphs_tags(tagged)`

형태소 분석 결과 문자열을 받아 `(형태소, 품사)` 튜플의 리스트를 반환합니다.

#### 파싱 알고리즘

형태소 분석 결과는 단순한 `+` split이나 마지막 `/` split으로는 처리할 수 없는 특이 케이스가 존재합니다.

**특이 케이스 1 — 형태소 자체가 `+`인 경우**

`++/품사` 패턴에서 앞의 `+`는 구분자이고, 뒤의 `+`가 형태소입니다.

```
BB/SL++/SW+이하/NNG+의/JKG
→ ('BB','SL'), ('+','SW'), ('이하','NNG'), ('의','JKG')
```

형태소가 `++`(두 개)인 경우도 동일한 원리로 처리합니다.
```
BB/SL+++/SW+이하/NNG
→ ('BB','SL'), ('++','SW'), ('이하','NNG')
```

**특이 케이스 2 — 형태소 자체가 `/`인 경우**

`//품사` 패턴에서 앞의 `/`가 형태소입니다.

```
강원/NNP+//SP+인제/NNP
→ ('강원','NNP'), ('/','SP'), ('인제','NNP')
```

**특이 케이스 3 — 형태소 내부에 `/`가 포함된 경우**

마지막 `/`만 품사 구분자이므로, 형태소 안에 `/`가 포함될 수 있습니다.

```
Supreme/Nike/NNP
→ ('Supreme/Nike', 'NNP')
```

#### 알고리즘 핵심 아이디어

`/품사태그` 패턴을 직접 탐색하되, **품사 태그 직후가 `+`(구분자) 또는 문자열 끝일 때만 유닛의 경계로 확정**합니다. 그렇지 않으면 형태소 내부의 `/`로 판단하고 탐색을 계속합니다.

```
현재 위치 i에서 시작
  └─ '/' 탐색
      ├─ '/' 뒤가 영문 대문자(품사 태그)?
      │    ├─ 품사 태그 끝이 '+' 또는 문자열 끝?
      │    │    └─ ✅ 유닛 확정: morph = tagged[i:slash], pos = 태그
      │    └─ 품사 태그 뒤에 다른 문자 있음?
      │         └─ 🔄 형태소 내부 '/', 다음 '/'부터 재탐색
      └─ '/' 뒤가 품사 태그 아님?
           └─ 🔄 다음 '/'부터 재탐색
```

## tagged2context.py 상세 설명

### 색인 대상 품사

| 품사 코드 | 의미 | 단일어 색인 | 복합어 참여 |
|----------|------|:-----------:|:-----------:|
| NNG | 일반명사 | ✅ | ✅ |
| NNP | 고유명사 | ✅ | ✅ |
| SH  | 한자 | ✅ | ✅ |
| SL  | 외국어(영어) | ✅ (단독 시에만) | ✅ |
| NR  | 수사 | ❌ | ✅ |
| NNB | 의존명사 | ❌ | ✅ |
| SN  | 숫자 | ❌ | ✅ |

> **SL 특이 규칙**: SL이 다른 색인 대상 형태소와 연속되어 복합어를 이룰 때는 단일어로 색인하지 않습니다.

### 핵심 함수: `get_index_terms(mt_list)`

`(형태소, 품사)` 튜플 리스트를 받아 색인어 리스트를 반환합니다.

#### 1단계 — 연속 그룹 분할

색인 대상 품사가 연속되는 구간을 하나의 그룹으로 묶습니다. 비대상 품사(`JKB`, `SS`, `VV` 등)를 만나면 그룹이 끊깁니다.

```
Y/SL + 2/SN + K/SL + 위원회/NNG + 에/JKB
         ↑ 색인 대상 4개 연속         ↑ 비대상(종료)
→ 그룹: [(Y,SL), (2,SN), (K,SL), (위원회,NNG)]

휴렛팩커드/NNP + (/SS + HP/SL + )/SS + 사/NNG
     ↑색인O    ↑끊김  ↑색인O  ↑끊김  ↑색인O
→ 그룹1: [(휴렛팩커드,NNP)]
→ 그룹2: [(HP,SL)]
→ 그룹3: [(사,NNG)]
```

#### 2단계 — 단일어 · 복합어 추출

각 그룹의 크기에 따라 규칙이 적용됩니다.

**그룹 크기 = 1 (단독 형태소)**

단일어만 추출합니다. SL은 복합어에 속하지 않으므로 단일어로 색인합니다.

```
KGB/SL (단독) → KGB
최근/NNG (단독) → 최근
```

**그룹 크기 = 2 (2개 연속)**

단일어를 먼저 출력하고, 복합어(결합형)를 그 뒤에 출력합니다.

```
SK/SL + 텔레콤/NNG
→ 단일어: 텔레콤   (SK는 SL이고 복합어에 속함 → 단일어 색인 안함)
→ 복합어: SK텔레콤
→ 출력:   텔레콤 SK텔레콤
```

**그룹 크기 ≥ 3 (3개 이상 연속)**

단일어를 먼저 출력하고, **가장 긴 복합어(전체 결합형) 하나만** 출력합니다. 중간 길이의 복합어는 출력하지 않습니다.

```
정부/NNG + 조직/NNG + 개편/NNG
→ 단일어: 정부, 조직, 개편
→ 복합어: 정부조직개편  (가장 긴 것만; '정부조직', '조직개편' 등은 출력 안함)
→ 출력:   정부 조직 개편 정부조직개편

Y/SL + 2/SN + K/SL + 위원회/NNG
→ 단일어: 위원회   (Y, K는 SL이고 복합어에 속함 → 단일어 색인 안함)
               (2는 SN → 단일어 색인 안함)
→ 복합어: Y2K위원회
→ 출력:   위원회 Y2K위원회

５/SN + 대/NNB
→ 단일어: 없음   (SN, NNB는 단일어 색인 안함)
→ 복합어: ５대
→ 출력:   ５대
```

## 의존 관계

`tagged2context.py`는 `get_morphs_tags.py`를 모듈로 import하여 사용합니다.
두 파일은 **반드시 같은 디렉터리**에 위치해야 합니다.

```python
import get_morphs_tags as mf
...
result = mf.get_morphs_tags(tagged.rstrip())
```
